1、Why TLog?
     经过封装，接口简单，调用接口即可实现日志加密打印。
2、Why TLog?
     加密后的字符长度和加密前的字符长度基本一致。
3、Why TLog?
     自带日志分片（目前设定10M切分）、自带日志清理。
4、Tlog支持的平台
     目前只支持Win7\8\10\11
5、Demo无法编译
     作者环境：VS2017，如果是lib无法引入，那么可以尝试动态加载TLog.dll进行调用（不会？自行百度搜索，或者交流群咨询
     后续会增加动态引入的接口）
6、其他语言调用？
     因为是标准C接口，其他语言也可以调用哦。
